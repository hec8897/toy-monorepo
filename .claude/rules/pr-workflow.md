# PR 생성 워크플로우 📋

PR 생성 전 **반드시** 작업 일지를 작성하고 커밋/푸시한 후에 PR을 생성합니다.
작업 일지의 목적은 Claude가 이전 작업 내용을 참고하여 더 정확한 도움을 제공하기 위함입니다.

---

## 📝 PR 생성 전 작업 일지 작성

### 1단계: 작업 일지 파일 생성

**위치**: `docs/` 폴더  
**파일명 형식**: `YYYY-MM-DD-{브랜치명}-{간단한-작업내용}.md`

```
예시:
docs/2026-04-04-feature-auth-login-jwt-구현.md
docs/2026-04-04-feature-web-dashboard-상품목록-추가.md
docs/2026-04-04-fix-api-결제-오류-수정.md
```

### 2단계: 작업 일지 템플릿

```markdown
# YYYY-MM-DD {브랜치명} 작업 일지

## 📋 작업 개요

- **브랜치**:
- **작업 일자**:
- **목적**:

## ✅ 완료된 작업

-

## 🔧 주요 변경사항

| 파일 | 변경 내용 |
| ---- | --------- |
| ...  | ...       |

## 🐛 발생한 문제 & 해결

-

## 💡 기술적 결정사항 (왜 이 방법을 선택했는가)

-

## 🔗 관련 이슈/참고

-
```

### 3단계: 커밋 & 푸시

```bash
git add docs/YYYY-MM-DD-{브랜치명}-{작업내용}.md
git commit -m "docs: {브랜치명} 작업 일지 추가"
git push
```

### 4단계: PR 생성

작업 일지 push 완료 후 PR 생성.

---

## ✅ PR 생성 전 체크리스트

- [ ] `docs/` 폴더에 작업 일지 파일 작성 완료
- [ ] 작업 일지 커밋 완료 (`docs: {브랜치명} 작업 일지 추가`)
- [ ] 원격 브랜치에 push 완료
- [ ] PR 생성

---

## ⚠️ 주의사항

- 작업 일지 없이 PR 생성 금지
- 파일명에 브랜치명을 포함하여 추적 가능하게 유지
- `docs/` 폴더의 파일은 삭제하지 않음 (Claude의 과거 작업 참고용)
