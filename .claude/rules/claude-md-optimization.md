# CLAUDE.md 최적화 가이드 📚

## 🎯 목적

CLAUDE.md 파일을 효율적으로 작성하여 Claude Code가 프로젝트를 빠르고 정확하게 이해하도록 돕습니다.

---

## 📐 구조 원칙

### 1. 계층적 구성

```
CLAUDE.md (핵심 정보만)
├── 프로젝트 개요 (1-2문장)
├── 기술 스택 (표 형식)
├── 필수 명령어 (자주 사용하는 것만)
├── 핵심 규칙 (5-10개)
└── 중요 경로 (주요 파일/폴더만)

.claude/rules/ (세부 규칙)
├── coding-standards.md
├── git-workflow.md
├── testing-guidelines.md
└── ...
```

### 2. 정보 분리 기준

| CLAUDE.md에 포함   | rules/에 분리      |
| ------------------ | ------------------ |
| 프로젝트 핵심 정보 | 상세 코딩 규칙     |
| 자주 쓰는 명령어   | Git Flow 전체 설명 |
| 필수 규칙 요약     | 테스트 가이드라인  |
| 폴더 구조 개요     | 스타일 가이드      |

## ✂️ 최적화 기법

### 1. 간결한 표현 사용

**❌ 비효율적:**

```markdown
## 개발 서버 실행 방법

백엔드와 프론트엔드를 동시에 실행하려면 다음 명령어를 사용합니다:
npm run dev

백엔드만 실행하고 싶다면:
npm run dev:backend

프론트엔드만 실행하고 싶다면:
npm run dev:frontend
```

**✅ 효율적:**

```markdown
## 명령어

| 용도         | 명령어                 |
| ------------ | ---------------------- |
| 전체 실행    | `npm run dev`          |
| 백엔드만     | `npm run dev:backend`  |
| 프론트엔드만 | `npm run dev:frontend` |
```

### 2. 중복 제거

- README.md와 중복되는 내용은 링크로 대체
- 외부 문서 (NestJS, Next.js 등) 설명은 제외
- 표준적인 규칙 (ESLint 기본 룰 등)은 생략

### 3. 컨텍스트 기반 정보

Claude가 **실제로 필요한** 정보만 포함:

- ✅ 프로젝트 특유의 규칙
- ✅ 비표준적인 설정
- ✅ 자주 실수하는 부분
- ❌ 일반적인 TypeScript 문법
- ❌ 프레임워크 기본 사용법
